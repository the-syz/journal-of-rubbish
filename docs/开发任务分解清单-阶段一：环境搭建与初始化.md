## 任务清单

### 任务1.1：项目初始化与目录结构创建

**任务标题**: 创建标准化项目目录结构

**任务描述**:
1. 按照设计文档中的项目结构规划，创建完整的目录树
2. 创建所有必要的空文件和占位文件
3. 初始化package.json（如果使用Node.js工具）
4. 创建.gitignore文件，排除不必要的文件
5. 创建基础配置文件（如.editorconfig, .prettierrc等）

**具体步骤**:
1. 在项目根目录创建以下目录结构：
   ```
   journal-of-rubbish/
   ├── index.html          # 首页（空文件）
   ├── about.html          # 期刊介绍页（空文件）
   ├── contact.html        # 联系我们页（空文件）
   ├── articles/           # 文章详情页目录
   │   └── .keep           # 占位文件
   ├── css/                # 样式文件目录
   │   ├── style.css       # 主样式（空文件）
   │   ├── responsive.css  # 响应式样式（空文件）
   │   └── modal.css       # 弹窗样式（空文件）
   ├── js/                 # JavaScript文件目录
   │   ├── main.js         # 主逻辑（空文件）
   │   ├── search.js       # 搜索功能（空文件）
   │   └── modal.js        # 弹窗功能（空文件）
   ├── data/               # 数据文件目录
   │   └── articles.json   # 文章数据（空文件）
   ├── assets/             # 静态资源目录
   │   ├── images/         # 图片资源目录
   │   │   └── .keep       # 占位文件
   │   └── fonts/          # 字体文件目录
   │       └── .keep       # 占位文件
   └── docs/               # 文档目录（已存在）
   ```
2. 创建package.json文件，包含基本项目信息：
   ```json
   {
     "name": "journal-of-rubbish",
     "version": "1.0.0",
     "description": "Journal of Rubbish - 虚假学术期刊投稿网站",
     "private": true
   }
   ```
3. 创建.gitignore文件，包含常见忽略规则：
   ```
   node_modules/
   .DS_Store
   Thumbs.db
   *.log
   .env
   .vscode/
   .idea/
   dist/
   build/
   ```

**依赖关系**: 无（初始任务）

**验收条件**:
1. 所有目录和空文件已按上述结构创建完成
2. 执行`dir /s`（Windows）或`tree`命令可看到完整目录树
3. package.json文件内容正确
4. .gitignore文件包含必要规则
5. 可在浏览器中直接打开index.html（显示空白页）

**预估工作量**: 0.5人/日

### 任务1.2：开发环境配置

**任务标题**: 配置开发工具和代码规范

**任务描述**:
1. 配置代码编辑器（VSCode）的开发环境
2. 设置代码格式化规范
3. 配置必要的开发工具扩展
4. 设置Git钩子（可选）

**具体步骤**:
1. 创建.editorconfig文件，统一编辑器配置：
   ```
   root = true
   
   [*]
   charset = utf-8
   indent_style = space
   indent_size = 2
   end_of_line = lf
   insert_final_newline = true
   trim_trailing_whitespace = true
   
   [*.html]
   indent_size = 2
   
   [*.css]
   indent_size = 2
   
   [*.js]
   indent_size = 2
   ```
2. 创建.prettierrc.json文件，配置代码格式化规则：
   ```json
   {
     "printWidth": 100,
     "tabWidth": 2,
     "useTabs": false,
     "semi": true,
     "singleQuote": true,
     "trailingComma": "es5",
     "bracketSpacing": true,
     "arrowParens": "avoid"
   }
   ```
3. 创建.vscode/extensions.json，推荐VSCode扩展：
   ```json
   {
     "recommendations": [
       "esbenp.prettier-vscode",
       "ritwickdey.LiveServer",
       "ms-vscode.live-server",
       "EditorConfig.EditorConfig"
     ]
   }
   ```
4. 创建.vscode/settings.json，配置工作区设置：
   ```json
   {
     "editor.formatOnSave": true,
     "editor.defaultFormatter": "esbenp.prettier-vscode",
     "files.autoSave": "afterDelay",
     "files.autoSaveDelay": 1000
   }
   ```
5. （可选）配置Git钩子，使用husky和lint-staged
   ```bash
   npm install --save-dev husky lint-staged
   npx husky install
   npx husky add .husky/pre-commit "npx lint-staged"
   ```
6. 在package.json中添加lint-staged配置：
   ```json
   {
     "lint-staged": {
       "*.{js,css,html}": ["prettier --write"]
     }
   }
   ```

**依赖关系**: 任务1.1（项目目录结构已创建）

**验收条件**:
1. .editorconfig、.prettierrc.json文件已创建且内容正确
2. VSCode配置目录已创建
3. 使用VSCode打开项目时，编辑器自动应用配置
4. 保存HTML/CSS/JS文件时自动格式化
5. （可选）Git提交前自动格式化代码

**预估工作量**: 0.5人/日

### 任务1.3：构建工具配置（可选）

**任务标题**: 配置现代化前端构建工具

**任务描述**:
1. 评估并选择构建工具（推荐Parcel，零配置）
2. 安装和配置构建工具
3. 配置开发服务器和热重载
4. 配置生产环境构建优化

**具体步骤**:
1. 安装Parcel构建工具：
   ```bash
   npm install --save-dev parcel
   ```
2. 更新package.json，添加scripts：
   ```json
   {
     "scripts": {
       "start": "parcel index.html",
       "build": "parcel build index.html about.html contact.html articles/*.html",
       "preview": "parcel preview"
     }
   }
   ```
3. 创建parcel配置（可选），在package.json中添加：
   ```json
   {
     "parcel": {
       "source": "src/",
       "distDir": "dist/",
       "publicUrl": "./",
       "cacheDir": ".parcel-cache"
     }
   }
   ```
4. 调整目录结构（如果使用Parcel）：
   - 将源代码移动到src/目录
   - 配置输出目录为dist/
5. 配置资源处理：
   - 确保图片、字体等资源正确引用
   - 配置CSS自动前缀
6. 创建基础HTML模板，包含必要的meta标签和资源引用

**依赖关系**: 任务1.1（package.json已创建）

**验收条件**:
1. 执行`npm install`成功安装依赖
2. 执行`npm start`成功启动开发服务器（默认http://localhost:1234）
3. 修改代码时浏览器自动刷新（热重载）
4. 执行`npm run build`成功生成优化后的生产代码
5. 构建产物在dist/目录中，且文件体积经过优化

**预估工作量**: 1人/日

### 任务1.4：基础HTML模板创建

**任务标题**: 创建网站基础HTML框架

**任务描述**:
1. 创建所有页面的基础HTML结构
2. 设置统一的meta标签和SEO基础信息
3. 引入基础CSS和JavaScript文件
4. 创建通用的页面模板结构

**具体步骤**:
1. 创建基础HTML模板（base-template.html）：
   ```html
   <!DOCTYPE html>
   <html lang="zh-CN">
   <head>
     <meta charset="UTF-8">
     <meta name="viewport" content="width=device-width, initial-scale=1.0">
     <meta name="description" content="Journal of Rubbish - 一个虚构的学术期刊投稿网站，仅供娱乐">
     <meta name="keywords" content="学术期刊,投稿,虚假期刊,娱乐网站">
     <title>Journal of Rubbish - 垃圾期刊</title>
     <link rel="stylesheet" href="/css/style.css">
     <link rel="stylesheet" href="/css/responsive.css">
     <link rel="stylesheet" href="/css/modal.css">
     <link rel="icon" href="/assets/images/favicon.ico" type="image/x-icon">
   </head>
   <body>
     <div class="page-container">
       <!-- 头部导航区域 -->
       <header class="site-header" id="siteHeader">
         <!-- 导航内容由JavaScript动态生成 -->
       </header>
       
       <!-- 主内容区域 -->
       <main class="main-content" id="mainContent">
         <!-- 页面特定内容 -->
       </main>
       
       <!-- 页脚区域 -->
       <footer class="site-footer" id="siteFooter">
         <!-- 页脚内容由JavaScript动态生成 -->
       </footer>
       
       <!-- 弹窗遮罩层 -->
       <div class="modal-overlay" id="modalOverlay" style="display: none;">
         <!-- 弹窗内容由JavaScript动态生成 -->
       </div>
     </div>
     
     <!-- JavaScript文件 -->
     <script src="/js/main.js" defer></script>
     <script src="/js/search.js" defer></script>
     <script src="/js/modal.js" defer></script>
   </body>
   </html>
   ```
2. 基于基础模板创建各个页面：
   - index.html（首页）
   - about.html（期刊介绍页）
   - contact.html（联系我们页）
   - articles/article-1.html 到 article-20.html（文章详情页）
3. 为每个页面设置特定的title和meta description
4. 创建基本的404页面（404.html）
5. 创建robots.txt文件，限制搜索引擎收录：
   ```
   User-agent: *
   Disallow: /
   ```

**依赖关系**: 任务1.1（目录结构已创建）

**验收条件**:
1. 所有HTML文件已创建且语法正确
2. 在浏览器中打开index.html能看到基本结构
3. 控制台无JavaScript错误
4. 页面标题和meta信息正确
5. 所有CSS和JS文件链接正确
6. robots.txt文件存在且内容正确

**预估工作量**: 1人/日

| 风险点 | 影响程度 | 应对措施 |
