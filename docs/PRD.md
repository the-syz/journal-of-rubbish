## 1. 项目概述

### 1.1 项目背景
《Journal of Rubbish》是一个虚构的学术期刊投稿网站项目，旨在通过模仿真实学术期刊的完整流程和界面，创建一个具有娱乐性质的展示性网站。该项目不涉及真实的学术投稿、审稿或出版功能，纯粹为娱乐目的而设计。

### 1.2 核心价值
- **娱乐展示**: 为朋友提供有趣、仿真的学术期刊网站体验
- **教育演示**: 展示学术期刊网站的基本结构和功能模块
- **技术实践**: 作为静态网站开发和部署的实践案例

### 1.3 目标用户画像
| 用户类型 | 特征描述 | 使用场景 |
|---------|---------|---------|
| 朋友观众 | 对学术期刊系统好奇，寻求娱乐体验 | 通过扫描二维码访问网站，浏览虚假文章，尝试交互功能 |
| 项目展示对象 | 技术爱好者，对网站实现感兴趣 | 查看网站设计、功能实现和技术细节 |

### 1.4 核心应用场景
1. **扫码访问**: 用户通过扫描二维码进入网站首页
2. **文章浏览**: 用户在首页浏览文章列表，点击进入详情页查看完整内容
3. **功能尝试**: 用户尝试点击各种入口按钮，体验交互反馈
4. **搜索查询**: 用户使用搜索功能查找感兴趣的文章

## 2. 功能需求清单

### 2.1 网站结构模块
| 功能点 | 触发条件 | 处理过程 | 预期结果 |
|--------|---------|---------|---------|
| 多页面导航 | 用户点击顶部导航栏 | 跳转到对应页面 | 显示目标页面内容 |
| 响应式布局 | 用户使用不同设备访问 | 根据屏幕尺寸自适应调整布局 | 在手机、平板、电脑上均能正常显示 |

### 2.2 首页功能模块
| 功能点 | 触发条件 | 处理过程 | 预期结果 |
|--------|---------|---------|---------|
| 顶部导航栏 | 页面加载 | 显示"首页"、"期刊介绍"、"联系我们"三个导航项 | 用户可点击导航到相应页面 |
| 搜索栏 | 用户输入关键词并提交 | 在前端进行文章数据匹配 | 显示匹配的文章列表（标题和作者匹配） |
| 期刊标签展示 | 页面加载 | 显示三个期刊标签："被打核心"、"垃科院一区silence"、"nothing" | 用户可看到期刊分类标识 |
| 文章推荐区 | 页面加载 | 显示精选的虚假文章列表（含标题、作者、摘要、发表日期） | 用户可浏览文章概要并点击查看详情 |
| 功能入口区 | 页面加载 | 显示"作者入口"、"编辑入口"、"专家入口"、"下载中心"按钮 | 用户可点击各入口按钮 |
| 公告栏 | 页面加载 | 显示网站公告信息 | 用户可查看最新通知 |

### 2.3 文章展示模块
| 功能点 | 触发条件 | 处理过程 | 预期结果 |
|--------|---------|---------|---------|
| 文章列表页 | 用户访问首页或点击导航 | 显示20篇虚假文章列表（分页或滚动加载） | 用户可浏览文章列表 |
| 文章详情页 | 用户点击文章标题或"阅读更多" | 加载并显示完整的文章内容 | 显示标准学术文章布局，包含：<br>1. 文章标题<br>2. 作者信息（姓名、单位）<br>3. 发表日期<br>4. 期刊标签<br>5. 摘要<br>6. 关键词<br>7. 正文内容（可使用乱码填充）<br>8. 参考文献<br>9. DOI号（虚假） |
| 文章标签筛选 | 用户点击期刊标签 | 筛选显示对应标签的文章 | 显示符合条件的文章列表 |

### 2.4 搜索功能模块
| 功能点 | 触发条件 | 处理过程 | 预期结果 |
|--------|---------|---------|---------|
| 标题搜索 | 用户在搜索框输入关键词 | 在前端数据中匹配文章标题 | 显示标题包含关键词的文章列表 |
| 作者搜索 | 用户在搜索框输入作者名 | 在前端数据中匹配文章作者 | 显示作者名包含关键词的文章列表 |
| 搜索结果展示 | 搜索执行后 | 以列表形式展示匹配结果 | 用户可点击搜索结果进入文章详情 |

### 2.5 交互功能模块
| 功能点 | 触发条件 | 处理过程 | 预期结果 |
|--------|---------|---------|---------|
| 作者入口弹窗 | 用户点击"作者入口"按钮 | 弹出模态对话框 | 显示提示信息："不给进"，并提供"确定"按钮 |
| 编辑入口弹窗 | 用户点击"编辑入口"按钮 | 弹出模态对话框 | 显示类似的拒绝访问提示 |
| 专家入口弹窗 | 用户点击"专家入口"按钮 | 弹出模态对话框 | 显示类似的拒绝访问提示 |
| 弹窗关闭 | 用户点击"确定"按钮或遮罩层 | 关闭模态对话框 | 返回原页面，可继续其他操作 |

### 2.6 静态页面模块
| 功能点 | 触发条件 | 处理过程 | 预期结果 |
|--------|---------|---------|---------|
| 期刊介绍页 | 用户点击"期刊介绍"导航 | 加载并显示期刊介绍内容 | 显示期刊的使命、范围、编委会等介绍信息（提供示例文字和编辑样式） |
| 联系我们页 | 用户点击"联系我们"导航 | 加载并显示联系信息 | 显示邮箱、联系人、电话等联系信息（提供示例文字和编辑样式） |

## 3. 非功能需求

### 3.1 性能指标
| 指标项 | 具体要求 | 备注 |
|--------|---------|------|
| 页面加载时间 | 首屏加载时间 ≤ 3秒 | 在常规网络环境下测试 |
| 搜索响应时间 | 前端搜索响应 ≤ 500毫秒 | 基于本地数据检索 |
| 并发访问 | 支持至少50个并发用户 | 静态网站，主要受托管平台限制 |

### 3.2 安全性要求
| 要求项 | 具体说明 |
|--------|---------|
| 无后端交互 | 网站为纯静态，不涉及用户数据提交或服务器交互 |
| 无敏感信息 | 所有内容均为公开虚假数据，不包含真实个人信息 |
| 跨平台安全 | 遵循标准HTML/CSS/JS安全实践，避免XSS等前端漏洞 |

### 3.3 兼容性要求
| 平台/环境 | 兼容要求 | 测试标准 |
|-----------|---------|---------|
| 浏览器 | Chrome 90+、Firefox 88+、Safari 14+、Edge 90+ | 核心功能正常，布局基本一致 |
| 设备类型 | 桌面电脑、平板、手机 | 响应式设计，适配不同屏幕尺寸 |
| 操作系统 | Windows 10/11、macOS 11+、iOS 14+、Android 10+ | 确保基础功能可用 |

### 3.4 可用性标准
| 标准项 | 具体要求 |
|--------|---------|
| 视觉设计 | 模仿正经学术期刊的严肃风格，采用灰色系配色方案 |
| 导航清晰 | 顶部导航栏始终可见，用户随时知道当前位置 |
| 交互反馈 | 所有可点击元素有明确的hover和active状态反馈 |
| 内容可读性 | 文字大小、行距、对比度符合无障碍阅读基本要求 |

## 4. 输入输出与接口规范

### 4.1 用户界面元素规范
| 元素类型 | 样式要求 | 交互状态 |
|---------|---------|---------|
| 导航栏 | 固定在顶部，灰色背景，白色文字，有hover效果 | 点击跳转，当前页高亮 |
| 按钮 | 学术风格设计，有边框和阴影，灰色系配色 | hover变深，active压下效果 |
| 搜索框 | 右侧带搜索图标，有placeholder提示 | 聚焦时有边框高亮 |
| 文章卡片 | 卡片式设计，有阴影和边框，包含文章基本信息 | 点击整个卡片可进入详情 |
| 弹窗 | 居中显示，有半透明遮罩层，有关闭机制 | 点击遮罩或确定按钮关闭 |

### 4.2 交互流程规范
1. **首页访问流程**:
   ```
   用户扫码 → 加载首页 → 显示导航栏/搜索/文章列表/功能入口 → 用户可进行各种操作
   ```

2. **文章查看流程**:
   ```
   用户点击文章卡片 → 跳转到文章详情页 → 显示完整文章内容 → 用户可返回首页
   ```

3. **搜索流程**:
   ```
   用户在搜索框输入关键词 → 点击搜索图标或按Enter → 显示搜索结果列表 → 用户可点击结果查看详情
   ```

4. **弹窗交互流程**:
   ```
   用户点击作者入口 → 显示"不给进"弹窗 → 用户点击确定按钮 → 弹窗关闭，返回原页面
   ```

### 4.3 数据格式规范
| 数据类型 | 格式要求 | 示例 |
|---------|---------|------|
| 文章数据 | JSON格式，包含完整文章信息 | 见附录：文章数据结构 |
| 配置数据 | JSON格式，包含网站基础配置 | { "siteTitle": "Journal of Rubbish", ... } |
| 静态内容 | Markdown或HTML格式，用于静态页面 | 期刊介绍、联系我们页面内容 |

### 4.4 外部接口规范
| 接口类型 | 规范说明 | 使用场景 |
|---------|---------|---------|
| GitHub Pages | 静态网站托管服务 | 网站部署和访问 |
| 二维码生成 | 第三方二维码生成服务 | 生成网站访问二维码 |
| 无后端API | 纯前端实现，无服务器交互 | 所有功能在前端完成 |

## 5. 验收标准

### 5.1 网站结构验收标准
| 验收项 | 验收条件 | 测试方法 |
|--------|---------|---------|
| 多页面导航 | 点击顶部导航栏各选项能正确跳转到对应页面 | 手动测试每个导航链接 |
| 响应式设计 | 在手机、平板、电脑三种设备尺寸下均正常显示 | 使用浏览器开发者工具测试不同分辨率 |

### 5.2 首页功能验收标准
| 验收项 | 验收条件 | 测试方法 |
|--------|---------|---------|
| 期刊标签展示 | 首页清晰显示三个标签："被打核心"、"垃科院一区silence"、"nothing" | 视觉检查标签存在且内容正确 |
| 文章推荐区 | 显示至少5篇推荐文章，每篇包含标题、作者、摘要、日期 | 检查文章数据加载和显示 |
| 功能入口按钮 | 显示"作者入口"、"编辑入口"、"专家入口"、"下载中心"四个按钮 | 检查按钮存在且可点击 |
| 公告栏 | 显示公告内容区域 | 检查公告区域存在且有内容 |

### 5.3 文章展示验收标准
| 验收项 | 验收条件 | 测试方法 |
|--------|---------|---------|
| 文章数量 | 网站包含20篇虚假文章 | 检查文章数据文件或数据库 |
| 文章详情页布局 | 包含标题、作者、日期、标签、摘要、正文、参考文献等标准元素 | 随机选择3篇文章检查详情页 |
| 文章标签筛选 | 点击期刊标签能筛选显示对应文章 | 测试每个标签的筛选功能 |

### 5.4 搜索功能验收标准
| 验收项 | 验收条件 | 测试方法 |
|--------|---------|---------|
| 标题搜索 | 输入文章标题关键词能匹配到对应文章 | 使用已知文章标题进行搜索测试 |
| 作者搜索 | 输入作者名能匹配到对应文章 | 使用已知作者名进行搜索测试 |
| 空结果处理 | 输入不存在的关键词显示无结果提示 | 测试不存在的搜索词 |

### 5.5 交互功能验收标准
| 验收项 | 验收条件 | 测试方法 |
|--------|---------|---------|
| 作者入口弹窗 | 点击"作者入口"弹出对话框，显示"不给进"和确定按钮 | 点击按钮检查弹窗内容和交互 |
| 弹窗关闭 | 点击确定按钮或遮罩层能关闭弹窗 | 测试两种关闭方式 |
| 其他入口弹窗 | 编辑入口和专家入口也有类似的弹窗交互 | 测试所有功能入口 |

### 5.6 静态页面验收标准
| 验收项 | 验收条件 | 测试方法 |
|--------|---------|---------|
| 期刊介绍页 | 页面加载正常，包含示例文字和正确样式 | 访问页面检查内容和样式 |
| 联系我们页 | 页面加载正常，包含联系信息示例 | 访问页面检查内容和样式 |

### 5.7 视觉设计验收标准
| 验收项 | 验收条件 | 测试方法 |
|--------|---------|---------|
| 灰色系配色 | 网站整体采用灰色系配色方案 | 视觉检查颜色使用 |
| 学术严肃风格 | 设计风格模仿正经学术期刊 | 对比真实学术期刊网站评估 |
| 整体美观度 | 布局合理，视觉层次清晰，无明显的样式问题 | 设计评审和用户测试 |
